# 系统架构

## 整体架构

```
┌─────────────────────────────────────────────────────────────┐
│                    微信小程序客户端                          │
├─────────────────────────────────────────────────────────────┤
│  顾客端页面层              │  老板端页面层                   │
│  - 首页                   │  - 数据看板                     │
│  - 服务列表               │  - 服务管理                     │
│  - 服务详情               │  - 订单管理                     │
│  - 订单列表               │  - 统计分析                     │
│  - 个人中心                                               │
└─────────────────────────────────────────────────────────────┘
                              ↓
┌─────────────────────────────────────────────────────────────┐
│                    微信云开发平台                            │
├─────────────┬─────────────┬─────────────┬─────────────────────┤
│   云函数     │  云数据库    │   云存储     │    微信生态服务     │
│ - 登录      │ - users     │ - 服务图片  │ - 微信登录         │
│ - 服务管理  │ - services  │ - 头像图片  │ - 订阅消息         │
│ - 预约管理  │ - schedules │             │                    │
│ - 订单管理  │ - orders    │             │                    │
│ - 会员管理  │ - members   │             │                    │
│ - 统计管理  │ - reviews   │             │                    │
└─────────────┴─────────────┴─────────────┴─────────────────────┘
```

## 数据架构

### 数据库集合

| 集合名 | 说明 | 主要字段 |
|--------|------|----------|
| users | 用户信息 | openid, nickname, avatar, role |
| services | 服务项目 | name, price, duration, image, status |
| schedules | 排班时段 | date, timeSlot, maxBookings, currentBookings |
| orders | 预约订单 | orderNo, userId, serviceId, status, amount |
| members | 会员信息 | userId, level, points, totalConsumption |
| reviews | 服务评价 | orderId, userId, rating, content |

## 业务流程

### 预约流程

```
用户浏览服务 → 选择服务 → 选择时间 → 确认预约 → 创建订单 → 预约成功
```

### 订单状态流转

```
pending（待确认） → confirmed (已确认) → completed (已完成)
                                         ↓
                                   cancelled（已取消）
```

## 安全机制

- 微信登录认证
- 云数据库权限控制
- 订单号唯一性保证
- 预约时间冲突检测
